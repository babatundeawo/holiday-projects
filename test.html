1. Executive Summary & Learning Outcomes

This capstone teaches students to design a durable, file-backed command-line Expense Tracker. The app captures transaction records, persists them in CSV, and produces human-readable reports. Students who complete this will demonstrate competency in file I/O, data structures, input validation, and user-driven program flow.

Learning outcomes

  • Design a menu-driven CLI application.
  • Persist structured data using CSV via Python's standard library.
  • Aggregate and format data for reports (category totals, listing).
  • Write modular code with clear function responsibilities and docstrings.
  • Package and document a submission for evaluation.

2. Prerequisites & Environment

Software

  • Python 3.8+ (any Python 3 release is acceptable). Verify with python --version or open IDLE and check Help → About.
  • IDLE or a code editor (VS Code recommended for experienced students; IDLE preferred for newcomers because F5 runs the script).

Hardware & files

  • Local folder for the project, e.g. ExpenseTracker/.
  • Write permission in that folder (so the program can create and update expenses.csv).
Instructor tip: Confirm students can run a trivial one-line script before assigning (e.g., a file with print("hello")).

4. Complete annotated sample code

This is the canonical sample students should be able to run and extend. Save it as expense_tracker.py. It's intentionally simple and uses only Python standard libraries.

# expense_tracker.py
import csv
from datetime import datetime
import os

FILENAME = "expenses.csv"

def initialize_file():
    """
    Ensure the CSV storage file exists and has a header row.
    This is idempotent — safe to call each time the app starts.
    """
    if not os.path.exists(FILENAME):
        with open(FILENAME, mode="w", newline="", encoding="utf-8") as file:
            writer = csv.writer(file)
            writer.writerow(["Date", "Category", "Description", "Amount"])

def add_expense():
    """
    Interactively collect expense details and append to the CSV.
    Minimal validation is performed; students can improve this.
    """
    date = datetime.now().strftime("%Y-%m-%d")
    category = input("Enter category (e.g., Food, Transport, Bills): ").strip().capitalize()
    description = input("Enter short description: ").strip()
    # Basic validation loop for numeric amount
    while True:
        amount_str = input("Enter amount spent (numbers only): ").strip().replace(",", "")
        try:
            amount = float(amount_str)
            break
        except ValueError:
            print("Invalid amount. Try again (e.g., 2500.50)")

    with open(FILENAME, mode="a", newline="", encoding="utf-8") as file:
        writer = csv.writer(file)
        writer.writerow([date, category, description, f"{amount:.2f}"])
    print("Expense recorded successfully!\\n")

def view_expenses():
    """
    Print all rows in the CSV in a human-readable table.
    """
    if not os.path.exists(FILENAME):
        print("No expenses recorded yet. Add an expense first.\\n")
        return

    with open(FILENAME, mode="r", encoding="utf-8") as file:
        reader = csv.reader(file)
        header = next(reader, None)
        print(f"\\n{header[0]:<12} {header[1]:<12} {header[2]:<30} {header[3]:>12}")
        print("-" * 72)
        for row in reader:
            print(f"{row[0]:<12} {row[1]:<12} {row[2]:<30} {row[3]:>12}")
        print()

def summarize_by_category():
    """
    Aggregate expenses by category and print totals.
    """
    totals = {}
    if not os.path.exists(FILENAME):
        print("No expenses recorded yet. Add an expense first.\\n")
        return

    with open(FILENAME, mode="r", encoding="utf-8") as file:
        reader = csv.DictReader(file)
        for row in reader:
            cat = row.get("Category", "Uncategorized")
            try:
                amt = float(row.get("Amount", 0))
            except ValueError:
                amt = 0.0
            totals[cat] = totals.get(cat, 0.0) + amt

    if not totals:
        print("No data to summarize.\\n")
        return

    print("\\nExpense Summary by Category:")
    print("-" * 35)
    for cat, total in sorted(totals.items(), key=lambda x: x[1], reverse=True):
        print(f"{cat:<20} ₦{total:,.2f}")
    print()

def main_menu():
    initialize_file()
    while True:
        print("Expense Tracker Menu")
        print("1. Add Expense")
        print("2. View All Expenses")
        print("3. Summary by Category")
        print("4. Exit")
        choice = input("Choose an option (1-4): ").strip()
        if choice == "1":
            add_expense()
        elif choice == "2":
            view_expenses()
        elif choice == "3":
            summarize_by_category()
        elif choice == "4":
            print("Goodbye!")
            break
        else:
            print("Invalid option. Try again.\\n")

if __name__ == '__main__':
    main_menu()

5. Step-by-step build walkthrough (function-by-function)

5.1 initialize_file()

Purpose: Create the CSV file and header row if the file is missing. Why? This simplifies subsequent reads/writes — code can assume the header exists and uses csv.DictReader or positional readers safely.

5.2 add_expense()

Flow:

  1. Collect inputs (category, description, and amount). Use strip() to remove accidental spaces.
  2. Validate amount with a small loop to prevent program-crashing exceptions.
  3. Append a formatted row to expenses.csv using csv.writer.
  4. Friendly confirmation message.

5.3 view_expenses()

Flow:

  1. Ensure the file exists; if not, instruct the user to add expenses first.
  2. Open the CSV and print the header and rows aligned into fixed-width columns for readability.

5.4 summarize_by_category()

Flow:

  1. Read CSV as DictReader for convenient column access.
  2. Aggregate with a dictionary keyed by category.
  3. Sort the totals descending and print a formatted summary (currency formatted).

5.5 main_menu()

Flow:

  1. Call initialize_file().
  2. Show the interactive menu in a loop until the user chooses to exit.
  3. Route choices to the functions above.

6. Testing checklist & sample demo script

Before submission, students should perform the following manual tests and record results in their README.

  1. Startup test: Run expense_tracker.py in IDLE (F5). Confirm it creates expenses.csv with headers.
  2. Add test: Add 3 different expenses from the menu. Confirm they appear in expenses.csv and the amounts are formatted to two decimals.
  3. View test: Select "View All Expenses" and confirm rows display correct dates and descriptions.
  4. Summary test: Use "Summary by Category" and verify totals add up to the sum of the amounts for each category.
  5. Edge cases: Try invalid input (letters for the amount) and confirm the program prompts to re-enter.

7. Submission instructions & grading rubric

Submission packaging

  1. Create folder named exactly: Firstname_Lastname_ExpenseTracker.
  2. Include these files:
    • expense_tracker.py — runnable main script.
    • expenses.csv — optional (if produced during testing).
    • README.md — includes run steps, implemented features, and short reflection (3–5 sentences).
    • evidence/ folder with screenshots (optional but recommended).
  3. Upload the folder to the repository under /submissions or submit via the classroom channel (per instructor).

Rubric (default)

  • Functionality (50%) — All required features work as described.
  • Code quality (20%) — Clear function decomposition, docstrings, variable names, and minimal duplication.
  • Usability (15%) — Friendly messages, input validation, and readable output.
  • Creativity & extensions (15%) — Any additional features (export CSV summaries, date filters, search by description).

8. Suggested extensions (challenge tasks)

  1. Filter by date range: Let the user enter start/end dates and show only expenses within the range.
  2. CSV export of summary: Write the summary dictionary to a new CSV (e.g., summary_2025-10.csv).
  3. Monthly totals and charts: Save totals per month and (optional) plot using matplotlib. Note: matplotlib is not standard library — list it as optional dependency.
  4. Search and edit: Allow searching description text and editing an existing row.
  5. Currency and localization: Use locale to format currency for different regions.

9. Troubleshooting & FAQs

Common problems

  • Script crashes on startup: Confirm Python 3.x is installed and the file is run with the correct interpreter.
  • Cannot create expenses.csv: Check folder write permissions and whether antivirus or cloud-sync (OneDrive) is blocking writes.
  • Amounts not added properly: Ensure non-numeric characters are removed from inputs (commas) and parsing uses float().

FAQ

  • Q: Can students use external libraries?
    A: For the core deliverable, no. Extensions may use matplotlib or pandas but students must declare dependencies in README.
  • Q: Should the CSV be included in submissions?
    A: Optional — if included, graders can immediately re-run tests without entering sample data.
